# 地图印记与校准探矿合同

## 目标

地图资源档案、生态危险、探索日志和奖励事务已经形成稳定基础，但上一阶段的地图差异仍主要表现为概率、文案和普通补给。本系统把地图特征推进为一条可玩的成长闭环：

```text
在地图特征条件下完成粗粒度勘探
→ 解锁“地图印记”里程碑
→ 原子领取该地图唯一地质材料
→ 在工作台升级简易探矿仪
→ 使用地图校准仪继续探索
→ 记录进入同一探索日志和存档
```

整个流程不增加矿物坐标透视，也不修改旧世界的资源生成结果。

## 共享地图身份

生产地图身份统一位于：

```text
src/world/map_profile_catalog.gd
```

共享合同提供：

- 五个稳定地图 ID；
- 玩家可见地图名称；
- ID 合法性检查；
- 保持产品顺序的地图列表。

探索日志和奖励注册表不再各自维护一份五地图常量。静态测试还会把共享目录与 `map_profiles.json`、资源档案、生态档案、探矿配置和奖励配置交叉比较。

## 地图印记

`data/exploration_journal.json` 使用 schema 2，新增：

```text
signature_finding
kind = profile_rule
```

每张地图拥有独立且可达的条件：

| 地图 | 标志条件 |
|---|---|
| 星辰大陆 | 中层或下层，并达到可观或富集 |
| 荒漠遗迹 | 浅层或中层，并达到可观或富集 |
| 极寒冰原 | 下层或深层，至少普通资源信号 |
| 天空群岛 | 浅层，至少普通资源信号 |
| 深渊世界 | 当前危险达到危险或极高 |

同一条规则中的非空条件必须全部满足。规则只读取已有探索记录，不重新扫描世界，也不保存第二份状态。

### 可达性约束

结构正确不等于玩法可完成。静态门禁会把资源档案的累计阈值转换为理论矿物比例，并与里程碑接受的密度门槛比较。

例如荒漠旧资源上限约为 6.75%，而“富集”门槛为 7.5%。因此荒漠规则不能只接受富集；本轮改为接受可观或富集，并限制在浅层或中层。以后任何地图规则超过生产资源上限，CI 会直接失败。

## 地图专属材料

完成地图印记后，奖励事务根据当前世界 ID 解析唯一材料：

| 地图 | 材料 |
|---|---|
| 星辰大陆 | 翠绿共鸣石 |
| 荒漠遗迹 | 遗迹日耀玻璃 |
| 极寒冰原 | 冰原寒心晶 |
| 天空群岛 | 浮岛风鸣晶 |
| 深渊世界 | 深渊余烬 |

材料约束：

- `category = material`；
- 最大堆叠 16；
- 不带 `block_id`；
- 不追加方块 numeric ID；
- 每次地图印记只领取 1 个；
- 领取仍使用原子背包事务；
- 背包满时保持 `claimable`，不会部分发放；
- 已领取状态随世界保存，重载不会重复生成。

这些材料不是死内容，每一种都用于对应校准仪配方。

## 校准探矿仪

`data/prospecting.json` 升级为 schema 2。旧 `prospecting_kit` 继续作为默认工具，同时新增五个地图校准档案。

| 工具 | 地图 | 水平半径 | 垂直半径 | 理论样本 | 主要用途 |
|---|---|---:|---:|---:|---|
| 简易探矿仪 | 通用 | 6 | 12 | 637 | 旧行为与旧配方兼容 |
| 翠绿校准探矿仪 | 星辰大陆 | 8 | 8 | 729 | 平原、河谷和林地下方的广域扫描 |
| 遗迹校准探矿仪 | 荒漠遗迹 | 7 | 10 | 704 | 扩大沙海遗迹周边覆盖 |
| 冰原校准探矿仪 | 极寒冰原 | 5 | 18 | 684 | 加深冰峰和厚雪层下方扫描 |
| 浮岛校准探矿仪 | 天空群岛 | 8 | 6 | 567 | 扩大浅层岛面，并降低有效岩层要求 |
| 深渊校准探矿仪 | 深渊世界 | 5 | 18 | 684 | 强化洞穴垂直扫描并适应断续岩层 |

全局硬上限为 768 个样本。Registry 会计算理论样本数，任何档案超过自己的 `max_samples` 或全局上限都会拒绝加载。

### 地图限制

校准仪只在对应地图工作。错误地图使用会返回：

```text
handled = true
success = false
reason = wrong_calibration
```

不会进入冷却，不会扫描世界，也不会创建探索记录。简易探矿仪仍可在全部地图使用。

### 输出边界

实时结果可以包含：

- 使用的工具和 calibration ID；
- 实际样本数；
- 扫描半径和预算诊断；
- 深度、密度、主矿物趋势；
- 当前危险；
- 粗粒度区块坐标。

仍然禁止：

```text
positions
ore_positions
coordinates
单个矿物方块坐标
自动寻路路径
```

工具诊断只存在于当前运行结果；探索存档继续使用 v3 白名单，不扩大持久攻击面。

## 制作合同

所有校准仪都在工作台制作，并消耗：

```text
简易探矿仪 ×1
对应地图材料 ×1
地图主题普通材料
```

制作通过 `InventoryService.transact_items()` 一次提交。背包空间不足或原料缺失时，简易探矿仪和地图材料都不会被部分消耗。

## 兼容性

保持不变：

- 世界 Seed；
- 矿物深度与概率；
- hash 和 salt；
- 方块 ID 与 numeric ID；
- 旧简易探矿仪物品 ID；
- 旧简易探矿仪配方；
- 探索存档版本 3；
- 已保存探索记录；
- 已领取奖励集合；
- 最多 64 条探索记录；
- 粗粒度坐标边界。

新增材料只属于物品目录，不修改世界方块目录或旧区块网格。

## 测试门禁

### 静态合同

- `validate_prospecting.ps1`：六工具、五地图映射、配方、预算和 capability；
- `validate_exploration_journal.ps1`：八里程碑、五地图规则和允许值；
- `validate_exploration_rewards.ps1`：地图唯一材料与完整奖励解析；
- `validate_map_signature_prospecting.ps1`：地图目录、资源可达性、材料、配方、工具和奖励的跨目录闭环。

### 领域回归

- `prospecting_regression.gd`：旧工具兼容、错误地图拒绝、匹配地图扫描和预算；
- `exploration_journal_regression.gd`：十组正反地图规则与共享名称；
- `exploration_milestone_reward_regression.gd`：profile-only 奖励、幂等和满背包保护；
- `map_signature_prospecting_regression.gd`：五地图端到端目录、规则、材料、制作和运行时扫描。

### 真实桌面

`map_signature_prospecting_desktop_acceptance.gd` 使用生产：

```text
GameScene
CreatureSpawner
ExplorationDangerService
ProspectingService
ExplorationJournalService
ExplorationMilestoneRewardService
CraftingService
InventoryService
SaveService
```

真实验证：

1. 深渊夜间附近敌对生物形成危险环境；
2. 鼠标右键完成基础勘探；
3. J 键打开日志并阻断游戏输入；
4. 鼠标点击领取深渊余烬；
5. 工作台原子制作深渊校准仪；
6. 再次鼠标右键执行垂直半径 18 的校准扫描；
7. 样本预算与坐标边界；
8. 正式保存、返回主菜单和完整重载；
9. claimed、工具和两条探索记录恢复且不重复；
10. 1024×576 截图和日志证据。
