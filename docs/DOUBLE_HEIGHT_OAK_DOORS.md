# 双格木门合同

## 目标

木门是由上下两个体素组成、可四向放置并可切换开关状态的结构型方块。视觉、碰撞、放置预览、交互、采集、掉落和保存必须共享同一状态合同。

## 持久状态

门状态直接编码在既有世界方块 ID 中，不新增 `doors` 或 `door_runtime` 存档领域。

```text
4 个朝向
× 2 个开关状态
× 2 个上下半状态
= 16 个持久方块变体
```

兼容规则：

- 旧 `oak_door` numeric ID 保持 23；
- `oak_door` 解释为南向、关闭、下半；
- 其他 15 个状态只追加到 `BLOCK_IDS` 末尾；
- 所有状态掉落和背包映射都回到 `oak_door`；
- 世界只保存上下两个方块 ID 与位置。

## 唯一状态所有者

`BlockDoorPolicy` 是纯策略，负责：

- 状态变体映射；
- 上下半识别；
- 开关状态识别；
- 朝向解析；
- 成对合法性；
- 下半/上半位置解析；
- 关闭与打开碰撞盒；
- 两格放置预览盒。

`BlockDoorInteractionService` 是运行事务所有者，负责：

- 上下两格原子放置；
- 右键成对开关；
- 任一半采集时成对拆除；
- 失败回滚；
- 运行诊断与玩家反馈。

## 原子放置

放置前必须满足：

```text
下半格为空
上半格为空
下方支撑为实体方块
上下两格均不与玩家身体相交
选中的背包槽仍有一扇门
```

提交顺序：

```text
写入下半
→ 写入上半
→ 消耗一件 oak_door
```

任何一步失败都会恢复两个世界格子，并且不会消费物品。

## 开关事务

右键命中任一半都先解析完整门对：

```text
下半与上半朝向一致
开关状态一致
半部角色正确
```

切换时上下半必须同时从关闭状态变为打开状态，或同时从打开状态变为关闭状态。第二次写入失败时，第一格恢复原状态。

## 几何与碰撞

每个持久半部使用一个厚度为 `0.125` 的轴对齐盒。

- 关闭时位于体素中轴，阻挡门洞；
- 打开时旋转到体素边缘，释放主要通行空间；
- 上下半使用相同朝向与开关状态；
- `VoxelChunk` 继续通过通用 Partial Geometry 管线同时构建视觉和碰撞；
- 预览使用同一 `BlockDoorPolicy` 几何。

## 采集与掉落

`BlockHarvestService` 在提交删除前调用结构服务：

```text
命中任一半
→ 解析完整门对
→ 删除下半
→ 删除上半
→ 只发放 oak_door ×1
```

删除第二格失败时恢复第一格。损坏旧世界中的孤立门残片可单独移除，防止永久不可清理。

## 运行诊断

瞬时诊断包括：

```text
placement_count
toggle_count
removal_count
rejection_count
last_result
```

这些计数按世界清零，不进入存档。

## 测试

永久验收包括：

- 旧 numeric ID 稳定；
- 16 个状态完整且唯一；
- 四方向关闭/打开配对；
- 两格预览、支撑和上半占用拒绝；
- 玩家上半重叠拒绝；
- 上下半放置失败回滚；
- 背包提交竞争回滚；
- 上下半开关失败回滚；
- 命中上半采集并只掉落一件；
- 真实鼠标放置、开门、关门与采集；
- 正式保存、返回菜单和完整重载；
- Windows Release 实际导出与启动。
