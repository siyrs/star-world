# 深渊精英生态与重击合同

## 目标

深渊世界已经拥有更高资源密度、昼夜敌对压力、洞穴和岩浆危险，但此前所有敌对个体仍然是同一种普通僵尸。本轮增加一个低频、高信号、可观察且可躲避的精英闭环：

```text
深渊夜间或下层/深层
→ 低概率出现深渊重击者
→ 长前摇和更大预警圈
→ 后退躲避或攻击打断
→ 单次重击
→ 深渊余烬掉落
→ 进入既有校准探矿路线
```

精英不是简单提高攻击频率，也不引入 Boss、复杂状态效果或独立行为树。

## 生产数据

### 生物档案

`data/creatures.json`：

| 字段 | 普通僵尸 | 深渊重击者 |
|---|---:|---:|
| 最大生命 | 20 | 28 |
| 移动速度 | 2.1 | 1.45 |
| 单次伤害 | 1 | 4 |
| 危险权重 | 1.0 | 2.0 |
| 精英标记 | 否 | 是 |

深渊重击者用更低移动速度交换更高单次威胁，避免成为仅靠追击速度制造压力的换皮敌人。

### 攻击档案

`data/hostile_attacks.json`：

| 字段 | 普通僵尸 | 深渊重击者 |
|---|---:|---:|
| 侦测范围 | 18 | 20 |
| 命中范围 | 1.65 | 2.2 |
| 前摇 | 0.8 秒 | 1.35 秒 |
| 成功冷却 | 5 秒 | 7 秒 |
| 取消恢复 | 0.6 秒 | 0.85 秒 |
| 预警圈倍率 | 1.05 | 1.2 |

精英的单次攻击更强，但前摇更长、冷却更慢、预警范围更明显，玩家有更充分的观察和躲避时间。

### 生态档案

`data/creature_ecology.json` 升级为 schema 2。物种条目可以定义：

```json
{
  "id": "abyss_brute",
  "weight": 1,
  "cap": 1,
  "condition_mode": "any",
  "phase_ids": ["night"],
  "max_player_y": 19
}
```

含义：

- 只有深渊生态包含该物种；
- 与普通僵尸的权重为 `1 : 9`；
- 世界中最多一只；
- 夜间 **或** 玩家位于 Y19 以下时可出现；
- 表层白昼不会出现。

## 通用生态能力

### 不再把敌对生物等同于僵尸

`CreatureSpawner` 以前使用：

```text
zombie group
species_id == "zombie"
```

来统计敌对数量和决定是否绑定玩家目标。这会阻止任何新敌对物种加入。

现在统一使用：

```text
hostile group
CreatureFactory.is_hostile_species()
```

普通僵尸和精英共同参与敌对上限、附近查询、清理和诊断。

### 条件物种选择

`CreatureEcologyPolicy` 先根据以下上下文过滤候选，再进行权重抽样：

```text
phase
player_y
species_counts
per-species cap
condition_mode all / any
```

规则仍是纯策略，不读取场景树、不生成随机数，也不修改 Spawner。

### Factory 目录完整性

`CreatureFactory` 现在双向验证：

- 每个 `creatures.json` 档案必须有生产脚本；
- 每个生产脚本必须有数据档案。

这样新增生物时不会出现“数据存在但无法生成”或“脚本存在但没有权威属性”的漂移。

## 危险度

附近敌对数量仍用于玩家可理解的实体计数，但危险分额外使用有界权重：

```text
普通僵尸 = 1.0
深渊重击者 = 2.0
```

因此一只精英不会伪装成两只实体，但会比一只普通敌人贡献更高压力。HUD 原因显示：

```text
附近精英敌对生物
```

权重只在附近半径内累加，单个生物最大权重受限，不增加世界采样预算。

## 玩家提示

瞄准精英时显示：

```text
精英 · 深渊重击者
精英重击蓄力 · 0.7 秒后攻击；后退或击退可打断
[鼠标左键] 攻击并打断
离开红色预警圈可躲避
```

`elite` 和 `danger_weight` 进入焦点 Snapshot，但不会包含导航坐标、路径或额外世界位置列表。

## 掉落与成长用途

深渊重击者固定掉落一枚：

```text
abyss_cinder / 深渊余烬
```

该材料已经用于制作 `abyss_prospecting_kit`，因此精英战斗提供另一条有用途的深渊材料来源，而不是新增死内容。

保持的原有路线：

```text
完成地图印记
→ 原子领取深渊余烬
→ 制作深渊校准探矿仪
```

新增路线：

```text
击败深渊重击者
→ 收集深渊余烬
→ 制作深渊校准探矿仪
```

两条来源共享同一个普通物品和配方，不增加方块 numeric ID，也不创建第二种重复材料。

## 兼容性

保持不变：

- 普通僵尸生命、速度、伤害和 0.8 秒前摇；
- 普通僵尸五秒冷却；
- 玩家重复敌对伤害保护；
- 世界 Seed、资源分布和生态地图 ID；
- 物品和方块 ID；
- 探索与奖励存档 schema；
- 生物实例和攻击前摇仍为瞬时状态，不写入世界存档。

`creature_ecology.json` 的 schema 2 只扩展生产配置；世界存档没有生态条目，因此旧世界不需要迁移。

## 测试门禁

### 静态

`validate_abyss_elite.ps1` 验证：

- 生物属性和速度/重击权衡；
- 唯一且有用途的深渊余烬掉落；
- 精英攻击比普通僵尸更长、更慢、更明显；
- 深渊限定、低权重、最多一只、夜间或深层条件；
- Factory 双向目录；
- Spawner 不再特判僵尸；
- 危险、焦点和提示接入；
- 全量测试入口。

### 领域

`abyss_elite_regression.gd` 覆盖：

- 五物种目录；
- 条件选择正反边界；
- 精英上限；
- 通用敌对目标绑定；
- 普通与精英压力；
- 长前摇零早期伤害；
- 躲避取消；
- 单次四点重击；
- 稳定伤害来源；
- 深渊余烬掉落；
- 危险原因和精英提示。

### 真实桌面

`abyss_elite_desktop_acceptance.gd` 使用生产世界、玩家、Spawner、Factory、战斗、危险、背包、掉落和保存服务，真实执行：

1. 创建深渊世界并切到夜间；
2. 生成深渊重击者；
3. 验证精英危险压力；
4. 显示重击预警与提示；
5. 真实按住 `S` 后退并零伤害躲避；
6. 回到范围承受一次四点重击；
7. 真实鼠标攻击打断并击败精英；
8. 收集深渊余烬；
9. 正式保存、返回菜单并完整重载；
10. 验证材料只恢复一次且瞬时精英实例不进入存档；
11. 输出 1024×576 截图和日志证据。
