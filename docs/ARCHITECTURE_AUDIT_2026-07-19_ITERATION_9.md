## 总体结论

项目已经具备可复用的敌对前摇、生态档案、区域危险和真实桌面战斗验收，但“生态”仍然隐含等同于“普通僵尸”。若直接添加精英，最容易出现以下错误做法：

```text
在 Spawner 新增 if map == abyss
→ 在 BaseCreature 新增 if species == elite
→ 把敌对数量继续写成 zombie 数量
→ 提高速度和伤害
→ 新增一个没有用途的精英材料
```

这会重新制造物种分支、配置漂移、不可读攻击和死内容。本轮先修复这些扩展边界，再加入第一只低频地图精英。

## 已发现并修复

### P0 · 敌对身份被硬编码为普通僵尸

#### 现状

`CreatureSpawner` 使用：

```gdscript
_count_group(&"zombie")
species_id == "zombie"
```

分别承担敌对上限和目标绑定。

#### 影响

- 新敌对物种不会绑定玩家；
- 新敌对物种不计入生态上限；
- 区域危险可能忽略它；
- 同一地图可能超过敌对预算；
- 后续每个物种继续增加特判。

#### 处置

统一为：

```text
hostile group
+ CreatureFactory.is_hostile_species()
```

所有敌对物种共享生成上限、玩家目标、附近查询、危险和清理合同。

### P0 · 生态档案无法表达低频、条件和上限

#### 现状

物种条目只有：

```json
{"id":"zombie","weight":1}
```

无法表达夜间、深度或最多一只。

#### 影响

若不扩展纯策略，精英只能通过 Spawner 地图分支实现，地图规则会再次从数据泄漏到运行代码。

#### 处置

`creature_ecology.json` 升级为 schema 2，条目支持：

```text
cap
condition_mode
phase_ids
min_player_y
max_player_y
```

`CreatureEcologyPolicy` 在权重抽样前完成纯过滤。当前精英使用：

```text
深渊限定
权重 1 / 普通僵尸 9
最多一只
夜间 OR Y19 以下
```

### P1 · 生物脚本与数据档案可以漂移

#### 现状

Factory 有静态脚本表和独立 JSON，但加载时没有双向校验。

#### 影响

可能出现：

- JSON 中有生物，但 Factory 无法创建；
- Factory 有脚本，但使用空档案和未审计后备值；
- CI 只校验数量，不校验映射。

#### 处置

Factory 现在验证：

```text
每个档案有脚本
每个脚本有档案
```

并提供统一物种目录和验证错误。

### P1 · 区域危险只按敌对数量计分

#### 现状

一只普通僵尸和一只精英对危险分贡献相同。

#### 影响

HUD 无法表达“敌人数量相同，但其中有一只重击精英”的实际威胁。

#### 处置

保留玩家可理解的 `hostile_count`，另增加有界：

```text
hostile_pressure
```

普通僵尸为 `1.0`，深渊重击者为 `2.0`。危险原因显示“附近精英敌对生物”。环境采样预算不变。

### P1 · 精英内容可能成为纯数值换皮

#### 风险

只增加生命、伤害和速度会让精英更难，但不一定更可读或更有策略性。

#### 处置

深渊重击者采用明确权衡：

- 生命从 20 增至 28；
- 速度从 2.1 降至 1.45；
- 前摇从 0.8 秒增至 1.35 秒；
- 冷却从 5 秒增至 7 秒；
- 预警区更大；
- 单次伤害为 4；
- 仍可后退躲避、击退或硬直打断。

威胁来自可观察的重击，而不是不可读的高速贴身连击。

### P1 · 精英掉落容易成为死内容

#### 风险

新增独立“精英核心”会扩大物品目录，但没有后续用途。

#### 处置

复用现有：

```text
abyss_cinder / 深渊余烬
```

它已经进入深渊校准探矿仪配方。精英战斗成为地图印记奖励之外的另一条材料来源，不增加方块 ID 或重复材料。


## 保持的兼容合同

- 普通僵尸属性和攻击时序不变；
- 玩家重复敌对伤害保护不变；
- 地图敌对总上限不变；
- 世界 Seed、资源分布、方块 ID 和物品 ID 不变；
- 旧深渊余烬和校准探矿配方不变；
- 世界存档没有生物实例或生态状态字段；
- 前摇、精英实例和生态选择都是瞬时状态；
- 旧世界无需迁移。
