## 主要发现

### P1 · `CreatureSpawner.map_id` 是无效状态

`CreatureSpawner` 长期存在 `map_id`，但生成逻辑仍然固定为：

```text
夜晚且僵尸未满 → 僵尸
否则动物未满 → 鸡/牛/猪随机
```

影响：

- 五张地图的生态实际上完全相同；
- 深渊、天空和冰原只有视觉与地形差异；
- 地图资源风险收益缺少敌对生态支撑；
- `map_id` 给调用者造成“已支持地图生态”的错误印象。

处置：新增数据驱动生态 Registry 和纯选择 Policy，生产 Spawner 真正按地图和昼夜运行。

### P1 · 玩家无法理解危险来源

此前玩家只能从生命损失或看见僵尸后推断风险，地图、深度、夜晚、洞穴和岩浆没有统一解释。

影响：

- 风险反馈滞后；
- 深渊的高资源收益没有清晰代价；
- 探矿记录只有资源，没有风险上下文；
- 后续精英生物与特殊掉落缺少统一风险入口。

处置：增加有界 `ExplorationDangerService`，HUD 显示危险等级、分数和原因，探矿记录保存扫描时风险。

### P1 · 探索记录缺少风险版本合同

探索存档 v1 只保存资源层级。直接在记录中追加动态字段而不迁移，会导致旧世界展示不一致。

处置：探索状态升级为 v2，旧记录统一补齐 `unknown` 风险字段，保持原记录顺序、去重和 64 条上限。

### P1 · 高频危险评估必须有预算

危险 HUD 若每帧大范围读取世界，会成为新的性能热点，尤其在区块流式与大型牧场同时运行时。

处置：

```text
0.75 秒刷新
125 理论样本
125 硬上限
不强制加载区块
不修改世界
```

静态门禁禁止未来把硬上限提高到 512 以上。

### 地图生态差异

- 星辰大陆：丰富被动生态，白天无自然敌对压力；
- 荒漠遗迹：被动上限较低，黄昏和夜间敌对概率较高；
- 极寒冰原：牛权重更高，夜间上限提高；
- 天空群岛：鸡占主要权重，敌对上限最低；
- 深渊世界：白天也可生成敌对生物，夜间上限最高。

### 危险度

组合：

- 地图基础分；
- 深度；
- 昼夜；
- 附近敌对数量；
- 岩浆样本；
- 洞穴开放比例。

HUD 不依赖精确坐标，也不会改变输入或暂停状态。

## 后续优化

### P1 · 探索日志与里程碑

当前发现已经包含资源和风险，下一步可安全构建只读日志：

- 首次发现地图；
- 首次富集区；
- 首次高危区；
- 同一区块风险变化；
- 不提供矿物精确位置。

### P1 · 敌对攻击前摇

危险度解决了区域风险解释，但单个僵尸攻击仍需要更明确的前摇、音效和命中窗口。

### P2 · 生态数量配置与设置解耦

当前地图档案是产品默认值。未来若增加难度设置，应使用倍率叠加，而不是直接改写生态 Registry。

### P2 · Reusable Workflow

专项工作流继续增长，应抽取统一的 strict import、领域脚本、桌面脚本和 artifact 上传模板。
