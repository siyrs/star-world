# 星的世界 · Star World

《星的世界》是一款使用 Godot 4 和 GDScript 开发的 Windows 第一人称 3D 像素沙盒游戏。玩家可以选择五种世界，探索随机地形，采集、合成、烧制、建造并在昼夜循环中生存。世界修改、玩家状态、背包、容器、机器、生存数据和新手引导进度都会持久保存。

完整产品规划与架构演进见：[PRODUCT_ROADMAP.md](docs/PRODUCT_ROADMAP.md)

## 已实现玩法

- 30 种体素方块，包括地形、水/岩浆、矿物、玻璃、台阶、楼梯和建筑材料。
- 16×16 水平分块、确定性 Seed、预算化渐进加载/卸载和稀疏修改存档。
- 五张地图：星辰大陆、荒漠遗迹、极寒冰原、天空群岛、深渊世界。
- 第一人称移动、跳跃、冲刺、重力和碰撞；左键破坏，右键交互/放置/食用。
- 武器攻击、动物与僵尸受伤、死亡掉落以及玩家专用拾取。
- 36 格背包、9 格快捷栏、62 种物品，支持堆叠、交换、快速装备和序列化。
- 随身合成与工作台共 35 个原子配方；工作台必须右键世界方块，不能通过 UI 绕过权限。
- 真实熔炉机器：原料、燃料、产出三槽，7 个烧制配方，持续进度、燃料余量、离线恢复和位置型存档。
- 熔炉支持煤炭、原木、木板和木棍；产出槽满时暂停，清空前不可拆除。
- 27 格箱子，可与玩家背包转移物品，内容随世界保存；非空箱子有拆除保护。
- 生命、饥饿、饱和、自然恢复、饥饿伤害、死亡和重生。
- 10 分钟默认昼夜循环，太阳、环境光和天空颜色随时间变化。
- 鸡、牛、猪和僵尸：程序化模型、漫游/追击 AI、伤害、死亡动画和种群回收。
- 主菜单、多存档、设置、HUD、背包、合成、熔炉、容器、真实暂停和死亡 UI。
- 持久化新手引导：移动、环顾、采集、放置、背包和合成；按 F1 临时隐藏。
- 准星上下文提示会说明采集、攻击、打开工位/机器/箱子、放置和食用操作。
- 有界即时反馈队列，成功、警告和错误不会重复刷屏；熔炉完成会提示产出。
- 统一视觉 token 与 Theme，规范字体层级、间距、状态色、按钮和信息卡。
- F3 运行诊断：帧时间、卡顿、区块队列、内存、节点、生物、掉落物、输入状态和自适应流式档位。
- 区块构建会根据持续帧压力在保守、受限、均衡和吞吐档之间调整；不会改写玩家视距。
- 带临时文件与备份恢复的可靠 JSON 保存。
- 无外部版权资产：生物模型和环境/方块/生物音效均在运行时程序化生成。

## 快速开始

1. 安装 Godot 4.3 或更高版本（项目 CI 使用 Godot 4.7）。
2. 在 Godot Project Manager 中导入根目录下的 `project.godot`。
3. 按 `F6/F5` 运行，或启动已导出的 `StarWorld.exe`。
4. 在主菜单选择“开始游戏”，选择地图、输入世界名和 Seed，然后创建世界。

命令行构建和 Windows 导出方法见 [BUILD.md](BUILD.md)，总体架构见 [ARCHITECTURE.md](ARCHITECTURE.md)，玩家体验合同见 [PLAYER_EXPERIENCE.md](PLAYER_EXPERIENCE.md)，熔炉机器合同见 [FURNACE_MACHINES.md](FURNACE_MACHINES.md)，方块交互扩展合同见 [BLOCK_INTERACTIONS.md](BLOCK_INTERACTIONS.md)，运行诊断与发行验收见 [RUNTIME_DIAGNOSTICS.md](RUNTIME_DIAGNOSTICS.md)，自适应区块预算见 [ADAPTIVE_STREAMING.md](ADAPTIVE_STREAMING.md)。

## 操作

| 操作 | 按键 |
|---|---|
| 移动 | `W` `A` `S` `D` |
| 跳跃 | `Space` |
| 冲刺 | `Shift` |
| 视角 | 鼠标移动 |
| 破坏方块/攻击 | 鼠标左键 |
| 打开工作台、熔炉、箱子 | 瞄准后鼠标右键 |
| 放置选中方块 | 在非交互表面鼠标右键 |
| 食用选中食物 | 鼠标右键 |
| 快捷栏 | `1` 至 `9` 或滚轮 |
| 背包 | `E` |
| 随身合成 | `C` |
| 隐藏/显示新手引导 | `F1` |
| 运行诊断 | `F3` |
| 关闭界面/暂停 | `Esc` |
| 快速保存 | `F5` |

右键优先处理面前的可交互方块，然后才尝试放置或食用。F1 只改变引导可见性；F3 面板是纯展示层。两者都不会改变鼠标捕获、暂停或玩家输入状态。
