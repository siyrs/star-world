# 星的世界 · Star World

《星的世界》是一款使用 Godot 4 和 GDScript 开发的 Windows 第一人称 3D 像素沙盒游戏。玩家可以选择五种世界，探索随机地形，按工具等级采集资源，合成、烧制、建造、装备武器防具、经营农田与牧场，并在昼夜循环中生存。世界修改、玩家状态、背包、工具与装备耐久、角色属性、农业、动物、容器、机器、生存数据和新手引导进度都会持久保存。

## 已实现玩法

- 45 种以上注册体素与作物阶段，包括地形、水/岩浆、矿物、玻璃、建筑材料、农田、机器和多阶段作物。
- 原创程序化 16×16 像素纹理：草皮/泥土分层、原木年轮与树皮、矿石嵌点、木板拼缝、砖缝、功能方块表面和透明 cutout；不复制第三方游戏资产。
- 所有区块共享确定性纹理图集、最近邻过滤与顶/侧/底方向明暗，保留既有高性能合批和流式预算。
- 16×16 水平分块、确定性 Seed、预算化渐进加载/卸载和稀疏修改存档。
- 五张地图：星辰大陆、荒漠遗迹、极寒冰原、天空群岛、深渊世界。
- 第一人称移动、跳跃、冲刺、重力和碰撞；按住左键采集方块，单击左键攻击生物，右键交互/放置/食用/耕种。
- 方块硬度、推荐工具、最低工具等级、错误工具速度和掉落资格均由数据驱动。
- 木、石、铁、钻石四档镐、斧、铲和锄；耐久随使用消耗，损坏后移除，剩余耐久随背包存档。
- 铲提高草、泥土、沙子、雪和耕地的采集速度；锄可把草地或泥土开垦为耕地。
- 高级矿物具有工具门槛：铁矿需要石镐，金矿和钻石矿需要铁镐或更高等级才能获得掉落。
- 采集进度面板、准星工具建议、预计耗时、掉落警告、快捷栏耐久百分比和精确 Tooltip。
- 小麦、胡萝卜和马铃薯，多阶段生长、灌溉、堆肥、位置型存档、有界离线生长、成熟收获和自动补种。
- 角色装备槽：主手、头盔、胸甲、护腿和靴子；右键武器/防具即可装备，点击装备槽可安全卸下。
- 数据驱动武器伤害、攻击冷却、击退、命中反馈、装备属性聚合、防具减伤、主手与防具耐久，以及实时角色属性摘要。
- 皮革与铁制四件套均可通过工作台制作；替换装备会把旧装备原子返回背包，背包已满时拒绝操作而不丢物品。
- 武器攻击、动物与僵尸受伤、死亡掉落以及玩家专用拾取。
- 36 格背包、9 格快捷栏、80 种以上物品，支持堆叠、交换、快速装备、metadata 和序列化。
- 随身合成与工作台提供原子配方；工作台必须右键世界方块，不能通过 UI 绕过权限。
- 真实熔炉机器：原料、燃料、产出三槽，持续进度、燃料余量、离线恢复和位置型存档。
- 27 格箱子，可与玩家背包转移物品，内容随世界保存；非空箱子有拆除保护。
- 修理台、床与安全重生点、动物繁殖/幼崽成长、饲料吸引和持久鸡蛋生产。
- 生命、饥饿、饱和、自然恢复、饥饿伤害、死亡和重生。
- 10 分钟默认昼夜循环，太阳、环境光和天空颜色随时间变化。
- 鸡、牛、猪和僵尸：程序化模型、漫游/追击 AI、伤害、死亡动画和种群回收。
- 主菜单、多存档、设置、HUD、角色/背包、合成、熔炉、容器、修理、真实暂停和死亡 UI。
- 持久化新手引导：移动、环顾、采集、放置、背包和合成；按 F1 临时隐藏。
- 精确体素命中和放置预览：白色目标框、绿色可放置幽灵格、红色拒绝原因，预览位置与右键提交一致。
- 准星上下文提示会说明采集、攻击、工具要求、开垦、播种、作物成熟度、工位/机器/箱子、放置和食用操作。
- 有界即时反馈队列，成功、警告和错误不会重复刷屏；工具/装备损坏、装备切换、耕种和熔炉完成都有明确反馈。
- 统一视觉 token 与 Theme，规范字体层级、间距、状态色、按钮和信息卡。
- F3 运行诊断：帧时间、卡顿、区块队列、内存、节点、生物、掉落物、输入状态和自适应流式档位。
- 区块构建会根据持续帧压力在保守、受限、均衡和吞吐档之间调整；不会改写玩家视距。
- 带临时文件与备份恢复的可靠 JSON 保存。
- 无外部版权资产：方块纹理、生物模型、环境/方块/生物音效均在运行时程序化生成。

## 快速开始

1. 安装 Godot 4.3 或更高版本（项目 CI 使用 Godot 4.7）。
2. 在 Godot Project Manager 中导入根目录下的 `project.godot`。
3. 按 `F6/F5` 运行，或启动已导出的 `StarWorld.exe`。
4. 在主菜单选择“开始游戏”，选择地图、输入世界名和 Seed，然后创建世界。

## 工具与采集

基础工具等级：

```text
木制 power 1
石制 power 2
铁制 power 3
钻石 power 4
```

代表性规则：

- 石头、圆石和煤矿：木镐或更高等级可正常掉落；
- 铁矿：石镐或更高等级；
- 金矿、钻石矿：铁镐或更高等级；
- 原木、木板、工作台和箱子：斧效率更高；
- 草、泥土、沙子、雪和耕地：铲效率更高；
- 草地或泥土：锄可转换为耕地；
- 石头采集后掉落圆石；
- 水、岩浆和基岩不能直接采集。

工具不足时，提示会在操作前说明掉落门槛。部分方块仍可被缓慢破坏，但不会错误产生受保护材料。对于有资格掉落的方块，如果背包空间不足，采集提交会被取消，世界方块保持原样。

耐久保存在物品槽 `metadata.durability` 中。旧存档中没有该字段的工具按满耐久读取，自定义名称等其他 metadata 不会被覆盖。

## 农业与食物

1. 获取小麦种子、胡萝卜或马铃薯。
2. 在工作台制作锄。
3. 手持锄右键上方为空的草方块或泥土，开垦为耕地。
4. 使用邻近水源或水桶湿润耕地。
5. 手持对应种植材料右键耕地播种。
6. 作物依次经历四个可见像素阶段；湿润耕地生长更快。
7. 右键成熟作物收获并自动补种；堆肥可以推进未成熟作物阶段。
8. 小麦可制作面包，马铃薯和鸡蛋等可通过熔炉加工。

作物关闭游戏后继续按有界离线时间生长，最多模拟六小时。背包没有空间时不会收获，成熟作物保持原样；播种写入失败时种植材料会退回。

## 装备、属性与防御

角色面板与背包共用 `E`：

```text
主手武器
头盔
胸甲
护腿
靴子

攻击 / 防御 / 预计减伤 / 移动速度 / 采集速度
```

当前伤害减免公式：

```text
减伤比例 = min(80%, 防御 / (防御 + 20))
最终伤害 = max(0.5, 原始伤害 × (1 - 减伤比例))
```

装备武器后，攻击读取最终角色属性并遵循数据驱动冷却；成功命中才消耗武器耐久。防具实际吸收伤害时会消耗耐久。旧存档缺少耐久 metadata 时按满耐久读取。

## 熔炉使用

1. 在工作台合成并放置熔炉。
2. 右键熔炉打开专用机器界面。
3. 点击背包中的可烧制原料，系统会放入原料槽。
4. 点击煤炭、木材或木棍，系统会放入燃料槽。
5. 关闭界面后熔炉仍会工作；世界暂停时机器同步停止。
6. 点击产出槽把成品取回背包。三个槽位清空后才可安全拆除熔炉。

## 世界存档

存档使用带 `save_version` 的 JSON，每个世界一个目录。Windows 默认位置：

```text
%APPDATA%\Godot\app_userdata\星的世界\worlds\<world-id>\world.json
```

存档包含地图类型和 Seed、稀疏方块修改、玩家位置和朝向、背包及工具 metadata、装备槽位、角色基础属性和持久 modifier、农业、动物、容器、机器、生命/饥饿、昼夜时间、床重生点和引导进度。方块纹理是确定性视觉数据，不进入世界存档。写入先落到临时文件，旧主文件保留为备份；正式文件损坏时会尝试恢复有效临时文件或上一版本。

## 当前边界

当前版本定位为可持续扩展的单人核心。水和岩浆仍为静态体素；水与玻璃暂时共用 Alpha Scissor 视觉合同，尚未拆分独立半透明 Surface；台阶、楼梯、门、床和栅栏仍使用简化体素几何；纹理尚未加入生物群系染色、法线图、独立发光通道或玩家资源包。多人和大型商业内容库不在当前阶段范围内。
