# 星的世界 · Star World

《星的世界》是一款使用 Godot 4 和 GDScript 开发的 Windows 第一人称 3D 像素沙盒游戏。玩家可以选择五种世界，探索随机地形，按工具等级采集资源，合成、烧制、建造、装备武器防具、经营农田与牧场，并在昼夜和地图危险中生存。世界修改、玩家状态、背包、装备、农业、动物、容器、机器、生存、探索发现、里程碑奖励和引导进度都会持久保存。

## 已实现玩法

- 五张地图：星辰大陆、荒漠遗迹、极寒冰原、天空群岛和深渊世界；每张地图拥有独立资源档案、生物生态、危险基础值和初次勘探奖励。
- 45 种以上注册体素与作物阶段，包括地形、水/岩浆、矿物、玻璃、建筑材料、农田、机器和多阶段作物。
- 原创程序化 16×16 像素纹理、共享确定性图集、最近邻过滤和顶/侧/底方向明暗；不复制第三方游戏资产。
- 16×16 水平分块、确定性 Seed、预算化渐进加载/卸载、自适应区块构建和稀疏修改存档。
- 第一人称移动、跳跃、冲刺、重力和碰撞；按住左键采集方块，单击左键攻击生物，右键交互、放置、食用、耕种或探矿。
- 数据驱动方块硬度、推荐工具、最低工具等级、错误工具速度和掉落资格。
- 木、石、铁和钻石四档镐、斧、铲和锄；耐久随使用消耗并随背包存档。
- 精确体素命中和放置预览：白色目标框、绿色可放置幽灵格和红色拒绝原因，预览与提交共用规则。
- 第一人称手持物、使用/挥动反馈和十阶段世界采集裂纹。
- 小麦、胡萝卜和马铃薯，多阶段生长、灌溉、堆肥、位置型存档、有界离线成长、成熟收获和自动补种。
- 主手、头盔、胸甲、护腿和靴子装备槽；数据驱动属性、防御减伤、攻击冷却、击退、硬直和装备耐久。
- 鸡、牛、猪和僵尸：程序化模型、地图权重、昼夜生成、漫游/追击、伤害、死亡动画、繁殖、幼崽成长和种群回收。
- 饲料吸引、持久鸡蛋生产、肉类加工、面包和农业食物链。
- 36 格背包和 9 格快捷栏，支持堆叠、交换、快速装备、metadata、序列化和原子多物品事务。
- 随身合成与工作台合成使用原子背包事务：失败不会先扣材料或公开中间状态。
- 真实熔炉机器：原料、燃料、产出三槽，持续进度、燃料余量、离线恢复和位置型存档。
- 27 格箱子、非空拆除保护、修理台、床和安全重生点。
- 生命、饥饿、饱和、自然恢复、饥饿伤害、死亡和重生。
- 10 分钟默认昼夜循环，太阳、环境光和天空颜色随时间变化。
- 地图、深度、昼夜、附近敌对生物、岩浆和洞穴共同形成实时区域危险分；HUD 显示风险等级和主要原因。
- 可制作的简易探矿仪，以最多 700 个样本返回深度、资源密度和主矿物的粗粒度趋势，不暴露方块级矿物坐标。
- 按 `J` 打开探索日志，查看稳定发现编号、世界日期、地图、区块、深度、资源趋势、危险上下文和七个探索里程碑。
- 七个探索里程碑均有可领取奖励；初次勘探按地图提供不同补给。背包满时奖励保持待领取，释放空间后可以重试。
- 里程碑奖励由独立事务服务写入背包；UI 不直接发放物品，重复点击不会重复领取。
- 探索记录最多持久化 64 条，同一区域重新扫描只更新原记录；已领取奖励拥有独立版本化状态。
- 主菜单、多存档、设置、HUD、角色/背包、合成、熔炉、容器、修理、探索日志、真实暂停和死亡 UI。
- 持久化新手引导与准星上下文提示；F1 临时隐藏引导，F3 显示运行诊断。
- 有界即时反馈队列，成功、警告和错误不会重复刷屏。
- 带临时文件、字段白名单、领域迁移和备份恢复的可靠 JSON 保存。
- 无外部版权资产：方块纹理、生物模型、环境/方块/生物音效均在运行时程序化生成。

## 快速开始

1. 安装 Godot 4.3 或更高版本；项目 CI 使用 Godot 4.7。
2. 在 Godot Project Manager 中导入根目录下的 `project.godot`。
3. 按 `F6/F5` 运行，或启动已导出的 `StarWorld.exe`。
4. 在主菜单选择“开始游戏”，选择地图、输入世界名和 Seed，然后创建世界。

## 探索、危险与奖励

1. 在工作台制作简易探矿仪。
2. 前往地表、洞穴或不同深度，手持探矿仪按鼠标右键。
3. 扫描结果只报告当前区块的深度层、资源密度和主矿物趋势，不返回方块级坐标。
4. HUD 持续显示当前区域危险等级；探矿记录会保存扫描时的危险上下文。
5. 按 `J` 查看最近发现和里程碑。
6. 完成里程碑后，日志卡片会显示“可领取”和奖励内容。
7. 点击领取后，系统先模拟整个奖励包是否能放入背包；只有全部可放入时才一次提交。
8. 背包满时不会放入部分物品，也不会消耗领取资格。释放空间后再次点击即可。
9. 已领取状态随世界保存；重新进入世界不会重复发放物品。

初次勘探的基础奖励为火把，并按地图增加补给：

| 地图 | 额外补给 |
|---|---|
| 星辰大陆 | 苹果 |
| 荒漠遗迹 | 玻璃 |
| 极寒冰原 | 雪块 |
| 天空群岛 | 木板 |
| 深渊世界 | 熟鸡肉 |

## 工具、采集与合成

基础工具等级：

```text
木制 power 1
石制 power 2
铁制 power 3
钻石 power 4
```

代表性规则：

- 石头、圆石和煤矿：木镐或更高等级；
- 铁矿：石镐或更高等级；
- 金矿和钻石矿：铁镐或更高等级；
- 原木、木板、工作台和箱子：斧效率更高；
- 草、泥土、沙子、雪和耕地：铲效率更高；
- 草地或泥土：锄可转换为耕地；
- 水、岩浆和基岩不能直接采集。

工具不足时，提示会在操作前说明掉落门槛。背包空间不足时，有资格的采集提交会被取消，世界方块保持原样。

合成现在通过统一多物品事务完成：系统先在背包副本中模拟原料移除和产出加入，成功后一次提交；空间或原料不足时真实背包完全不变。

## 农业、装备与机器

农业流程：开垦耕地、邻近水源或水桶湿润、播种、等待四个可见阶段、收获并自动补种。作物离线成长最多模拟六小时，背包满时成熟作物保持原样。

角色面板与背包共用 `E`，展示主手、防具、攻击、防御、预计减伤、移动和采集速度。成功命中才消耗武器耐久，防具实际吸收伤害时才消耗耐久。

熔炉拥有独立原料、燃料和产出槽，关闭界面后继续工作，世界暂停时同步停止。三个槽位清空后才可安全拆除。

## 世界存档

存档使用带 `save_version` 的 JSON，每个世界一个目录。Windows 默认位置：

```text
%APPDATA%\Godot\app_userdata\星的世界\worlds\<world-id>\world.json
```

存档包含地图和 Seed、稀疏方块修改、玩家位置、背包及 metadata、装备和属性、农业、动物、容器、机器、生命/饥饿、昼夜、床重生点、引导、探索记录和已领取里程碑奖励。新世界从创建时就拥有 `exploration.version = 3` 和 `exploration_rewards.version = 1`；旧世界加载时会补齐并规范化这些领域状态。

写入先落到临时文件，旧主文件保留为备份；正式文件损坏时会尝试恢复有效临时文件或上一版本。

2. 全量 Runtime 与领域回归；
3. 真实桌面按钮、鼠标、机器、采集、装备、农业、探索、满背包奖励重试和完整世界重载；
4. 实际 Windows Release 导出、启动、画面和退出资源检查。

## 当前边界

当前版本定位为可持续扩展的单人核心。水和岩浆仍为静态体素；水与玻璃暂时共用 Alpha Scissor 视觉合同；部分建筑交互仍为简化体素规则。地图特有材料、敌对攻击前摇、少量精英变体、共享 Machine Base、ServiceHub 生命周期组合化和 GitHub Actions reusable workflow 是后续重点。多人和大型商业内容库不在当前阶段范围内。
